## 2024-03-19 - Safe threadpool offloading of sqlite3 in FastAPI
**Learning:** Offloading `persistence.*` operations to thread pools via `asyncio.to_thread` is only safe because `backend/persistence.py` instantiates the sqlite3 connection with `check_same_thread=False` and guards access with a global lock `_LOCK = threading.Lock()`. If this lock or flag was missing, the offloading would result in a `sqlite3.ProgrammingError` and catastrophic database failure due to SQLite connections generally being tied to the creating thread.
**Action:** When migrating synchronous I/O or SQLite database operations to async threadpools, always explicitly verify that the underlying connection pool or library configuration (like `check_same_thread=False`) supports cross-thread use and handles concurrency safely, else the "optimization" becomes a runtime crasher.
